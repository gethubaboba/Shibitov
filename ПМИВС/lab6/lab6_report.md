## 2. Выполненные задания

### Задание 7.1 - SharedPreferences

**Проект:** `Lab6_Task71_SharedPreferences_Shibitov`

Реализовано приложение "Угадай число" с сохранением данных в двух файлах `SharedPreferences`.

Файл `app_settings` используется для хранения служебной информации:

- уникальный идентификатор приложения;
- количество запусков;
- лучший результат игры;
- суммарное число попыток.

Файл `user_profile` используется для хранения данных пользователя:

- адрес электронной почты;
- пароль.

Приложение состоит из экрана авторизации и игрового экрана. При первом входе пользователь вводит данные, после чего они сохраняются локально. При последующих запусках приложение может использовать уже сохраненные значения. На игровом экране отображается статистика, которая обновляется после прохождения игры.

### Задание 7.2 - Preferences DataStore

**Проект:** `Lab6_Task72_DataStore_Shibitov`

Реализована версия приложения "Угадай число", в которой вместо `SharedPreferences` используется `Preferences DataStore`.

Для работы с данными выделены репозитории:

- `AppSettingsRepository` - хранит настройки приложения и игровую статистику;
- `UserProfileRepository` - хранит данные профиля пользователя.

DataStore выбран как более современный способ хранения настроек: операции чтения и записи выполняются асинхронно через Kotlin coroutines и `Flow`, поэтому они не блокируют главный поток приложения.

### Задание 7.4 - SQLite

**Проект:** `Lab6_Task74_SQLite_Shibitov`

Реализовано приложение для работы с таблицей книг через SQLite и класс `SQLiteOpenHelper`.

Структура таблицы `book`:

| Поле | Тип | Назначение |
|---|---|---|
| `_id` | INTEGER PRIMARY KEY | Идентификатор записи |
| `author` | TEXT | Автор книги |
| `title` | TEXT | Название книги |
| `year` | INTEGER | Год издания |

Приложение создает базу данных, заполняет ее начальными данными, позволяет просматривать список книг, добавлять новую запись и изменять существующую запись. Для выполнения операций используются SQL-запросы и стандартные средства Android SQLite.

### Задание 7.5 - Обновление схемы SQLite

**Проект:** `Lab6_Task75_SQLite_Upgrade_Shibitov`

В этом задании расширена структура базы данных из задания 7.4. В таблицу `book` добавлены новые поля:

| Поле | Тип | Назначение |
|---|---|---|
| `publisher` | TEXT | Издательство |
| `pages` | INTEGER | Количество страниц |
| `price` | INTEGER | Стоимость |

Обновление схемы выполняется в методе `onUpgrade()`. Для сохранения уже существующих данных используются команды `ALTER TABLE`, а не полное удаление и повторное создание таблицы.

Пример логики обновления:

```kotlin
override fun onUpgrade(db: SQLiteDatabase, oldVersion: Int, newVersion: Int) {
    if (oldVersion < 2) {
        db.execSQL("ALTER TABLE book ADD COLUMN publisher TEXT NOT NULL DEFAULT ''")
        db.execSQL("ALTER TABLE book ADD COLUMN pages INTEGER NOT NULL DEFAULT 0")
        db.execSQL("ALTER TABLE book ADD COLUMN price INTEGER NOT NULL DEFAULT 0")
    }
}
```

Такой подход позволяет изменять структуру базы данных без потери информации, которая была сохранена пользователем ранее.

### Задание 7.7 - Android Room

**Проект:** `Lab6_Task77_Room_Shibitov`

Реализовано приложение с базой данных "Фильмотека" на основе библиотеки Room. Room используется как ORM-слой над SQLite и позволяет описывать структуру данных через Kotlin-классы и аннотации.

В базе данных используются две связанные таблицы:

| Таблица | Назначение |
|---|---|
| `film` | Хранение фильмов |
| `session` | Хранение сеансов, связанных с фильмами |

Таблица `film` содержит название фильма, режиссера, год выпуска и жанр. Таблица `session` хранит дату сеанса, количество зрителей, цену билета и ссылку на фильм.

В приложении реализованы три вкладки:

- "Фильмы" - добавление фильмов, сортировка, фильтрация и группировка по жанрам;
- "Сеансы" - добавление и просмотр сеансов;
- "Статистика" - расчет агрегированных показателей по фильмам.

Для получения статистики используется SQL-запрос с `JOIN`, `COUNT`, `SUM`, `AVG`, `MAX` и `MIN`. Room проверяет запросы на этапе компиляции и генерирует вспомогательный код через KSP.

## 3. Контрольные вопросы

**Что такое SharedPreferences?**  
`SharedPreferences` - это механизм Android для хранения небольших наборов данных в формате ключ-значение. Он подходит для настроек, флагов, счетчиков и простых пользовательских параметров.

**Чем DataStore отличается от SharedPreferences?**  
DataStore работает асинхронно, использует Kotlin Flow и лучше подходит для современных Android-приложений, так как не блокирует главный поток и обеспечивает более надежную обработку данных.

**Для чего нужен SQLiteOpenHelper?**  
`SQLiteOpenHelper` помогает создавать, открывать и обновлять SQLite-базу данных. В нем переопределяются методы `onCreate()` и `onUpgrade()`.

**Как обновить схему базы данных без потери данных?**  
Нужно увеличить версию базы данных и в `onUpgrade()` выполнить миграцию, например через `ALTER TABLE`, сохранив существующие записи.

**Что такое Room?**  
Room - это библиотека Jetpack для удобной работы с SQLite. Она позволяет описывать таблицы как `@Entity`, запросы как методы `@Dao`, а базу данных как класс `@Database`.
