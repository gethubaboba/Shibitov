## Задание 2. Разработка простого приложения

**Проект:** `Task2TestApp`

Разработано простое приложение на основе базового примера. Приложение демонстрирует работу с интерфейсом на QML: содержит текстовые блоки и интерактивные кнопки, по нажатию на которые динамически изменяется текст.

### Ключевые фрагменты кода

```qml
Text {
    id: messageText
    text: "Приложение создано с использованием Qt Creator и QML для Android."
    font.pixelSize: 17
    color: "#333333"
    horizontalAlignment: Text.AlignHCenter
    wrapMode: Text.WordWrap
    Layout.fillWidth: true
}

Button {
    id: actionButton
    text: "Изменить текст"
    Layout.fillWidth: true

    onClicked: {
        messageText.text = "Кнопка нажата. Интерфейс реагирует на действие пользователя."
    }
}
```

## Задание 4. Разработать приложение «Угадай число»

**Проект:** `Task4GuessNumber`

Разработано приложение «Угадай число», демонстрирующее более сложную бизнес-логику и работу с настройками.

**Реализовано:**
- Архитектура приложения строится по паттерну MVVM (`gameViewModel` и `gameSettings`).
- Приложение поддерживает локализацию на три языка (Русский, English, Беларуская) через встроенный `ComboBox`.
- Настройки (количество попыток, количество запусков, лучший результат) сохраняются между сессиями.
- Добавлена визуальная анимация (`NumberAnimation`) при нажатии кнопки "Проверить", создающая эффект пульсации.

### Ключевые фрагменты кода

```qml
ComboBox {
    Layout.fillWidth: true
    textRole: "name"
    valueRole: "code"
    model: [
        { name: "Русский", code: "ru" },
        { name: "English", code: "en" },
        { name: "Беларуская", code: "be" }
    ]
    onActivated: languageManager.setLanguage(currentValue)
}

NumberAnimation {
    id: pulseAnimation
    target: checkButton
    property: "scale"
    from: 1.0
    to: 1.06
    duration: 120
    easing.type: Easing.InOutQuad
    onFinished: checkButton.scale = 1.0
}
```

## Контрольные вопросы

**1. Опишите основные этапы установки и развертывания Aurora SDK?**
Скачивание установщика (онлайн или офлайн), установка Aurora IDE, настройка и запуск виртуальных машин для Build Engine и Эмулятора, авторизация, загрузка примеров проектов.

**2. Какую роль выполняет Build Engine в Aurora SDK?**
Предоставляет изолированное окружение сборки, не зависящее от хостовой ОС, позволяя кросс-компилировать проекты под различные архитектуры целевых устройств ОС Аврора.

**3. Как создать новый проект в Aurora IDE для ОС Аврора?**
В меню `File` выбрать `New File or Project`, затем `Aurora OS` -> `Application`. Задать имя, путь, выбрать необходимые комплекты сборки и завершить работу мастера.

**4. Как организовать взаимодействие между C++ и QML?**
Путем регистрации C++ типа с помощью `qmlRegisterType` или через передачу объекта в корневой контекст `QQmlContext::setContextProperty()`. В C++ используются макросы `Q_OBJECT`, `Q_PROPERTY`, `Q_INVOKABLE` для свойств и методов, доступных в QML.

**5. Приведите пример создания форматированного текста на языке QML?**
```qml
Text {
    text: "<b>Жирный</b> и <i>курсивный</i> текст"
    textFormat: Text.RichText
}
```

**6. Приведите примеры TextField и обработки ввода.**
```qml
TextField {
    placeholderText: "Введите имя"
    onTextChanged: console.log("Текст изменен на: " + text)
    onEditingFinished: console.log("Ввод завершен: " + text)
}
```

**7. Как добавить и использовать ресурсы (изображения, шрифты) в проекте Aurora OS?**
Необходимо создать и добавить файлы ресурсов в файл `.qrc`. Доступ в QML осуществляется через префикс `qrc:/` (например, `source: "qrc:/images/icon.png"`).

**8. Как обработать нажатие кнопки в QML и вызвать C++ метод?**
```qml
Button {
    text: "Нажми меня"
    onClicked: myCppObject.someMethod() // someMethod должен быть Q_INVOKABLE
}
```

**9. Как настроить и использовать систему локализации (переводов) в приложении Aurora OS?**
Необходимо обернуть строки в QML в функцию `qsTr()`, сгенерировать `.ts` файлы с помощью `lupdate`, перевести в Qt Linguist, скомпилировать `.qm` файлы с помощью `lrelease` и загрузить нужный файл с помощью класса `QTranslator` в C++.

**10. Приведите пример анимации элементов на языке QML.**

}
```

```

**13. Охарактеризуйте QTObject.**
Это самый базовый тип в QML (обертка над `QObject`), который не имеет визуального представления. Он часто используется для создания структур данных или группировки свойств.

**14. Перечислите особенности сигналов и слотов.**
- Слабая связность (объекту не нужно знать о существовании другого объекта).
- Типобезопасность (совпадение сигнатур проверяется на этапе компиляции).
- Возможность кросс-потокового взаимодействия.

**15. Как работать с файловой системой в приложении Aurora OS (чтение/запись файлов)?**
В C++ используются классы `QFile`, `QTextStream` и `QDir`. Для определения корректных путей для записи (например, директории документов или кэша) рекомендуется использовать `QStandardPaths`.

**16. Как отобразить список данных в QML с использованием модели из C++?**
Создать класс, наследующий `QAbstractListModel`, реализовать методы `rowCount()`, `data()` и `roleNames()`. Зарегистрировать модель в QML и передать ее свойству `model` компонента `ListView` или `SilicaListView`.

**17. Какой метод используется для выполнения SQL-запросов в мобильном приложении для ОС Аврора?**
Для выполнения запросов используется метод `exec()` класса `QSqlQuery` (например, `query.exec("SELECT * FROM table")`).

**18. Как обработать жизненный цикл приложения (пауза, возобновление) в Aurora OS?**
Через подключение к сигналам `QGuiApplication::applicationStateChanged` в C++ или мониторинг `Qt.application.state` в QML (свойства `Qt.ApplicationActive`, `Qt.ApplicationSuspended` и т.д.).

**19. Как выполнить HTTP-запрос из приложения Aurora OS и обработать ответ в QML?**
Лучшая практика: использовать класс `QNetworkAccessManager` в C++, отправлять запросы (GET/POST), читать ответ в слоте, привязанном к сигналу `finished`, и передавать распарсенные данные (например, JSON) в QML через сигналы и свойства. 

**20. Как собрать и опубликовать пакет RPM для ОС Аврора?**
Сборка RPM пакета для установки на устройства ОС Аврора осуществляется автоматически с помощью Aurora OS Build Engine на основе спецификаций `.spec` файла проекта при запуске развертывания проекта в Aurora IDE.
