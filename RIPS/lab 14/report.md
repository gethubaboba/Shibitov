# Лабораторная работа 14

## Описание проекта

Проект реализует распределенную систему хранения данных на основе Java sockets.
Каждый узел хранит одну пару `ключ-значение` и список соседних узлов. Клиент может
подключиться к любому узлу и выполнить запрос `GET <key>`.

Если ключ находится на текущем узле, сервер сразу возвращает ответ `FOUND`.
Если ключ отсутствует, сервер пересылает запрос соседним узлам в соответствии с
выбранной топологией. Для защиты от циклов в запросе передается список уже
посещенных портов.

Реализованы три топологии:

- `full`: каждый узел связан с каждым;
- `star`: первый узел является центральным;
- `tree`: узлы соединены как бинарное дерево.

## Текст программы

Исходный код расположен в каталоге `src`:

- `NodeServer.java`: сервер узла распределенного хранилища;
- `KeyValueClient.java`: клиент для запроса значения по ключу;
- `Topology.java`: построение списков соседей для разных топологий;
- `DemoLauncher.java`: демонстрационный запуск нескольких узлов и клиентов.

## Использованные источники

- Java Platform, Standard Edition Documentation: `java.net.ServerSocket`, `java.net.Socket`.
- cppreference / Java socket examples were used only as conceptual references for client-server interaction.
- AI assistant Codex was used to draft and verify the implementation.
