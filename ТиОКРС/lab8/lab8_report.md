## 2. Теоретические сведения

### 2.1. Метрики сложности потока управления

**Цикломатическая сложность Маккейба (V(G))** — наиболее широко применяемая метрика сложности потока управления.

*Граф потока управления (CFG)* — ориентированный граф G = (N, E), где:
- N — множество узлов (базовых блоков),
- E — множество рёбер (переходов между блоками).

**Формула:**
```
V(G) = E − N + 2·P
```
где P — число компонент связности (для одной функции P = 1).

**Альтернативная формула** (через предикатные узлы):
```
V(G) = π + 1
```
где π — число предикатных узлов (точек ветвления: `if`, `else if`, `for`, `while`, `case`, `&&`, `||`).

**Интерпретация V(G):**

| V(G) | Уровень риска |
|------|--------------|
| 1–10 | Простой, низкий риск |
| 11–20 | Умеренный риск |
| 21–50 | Высокий риск |
| > 50 | Очень высокий риск (рефакторинг рекомендован) |

**Существенная сложность (EC)** — цикломатическая сложность после редукции хорошо структурированных конструкций:
```
EC(G) = V(G) − m
```
где m — число редуцированных подграфов (правильно вложенных if/for/while).

### 2.2. Метрики сложности потока данных

**Определения:**
- **def(v)** — точка определения переменной v (присваивание).
- **c-use(v)** — вычислительное использование v (в арифметическом/строковом выражении).
- **p-use(v)** — предикатное использование v (в условии ветвления или цикла).
- **Пара def-use (DU-пара)** — пара (определение d, использование u) такая, что существует путь от d до u, на котором v не переопределяется.

**Метрика Шапена (Chapin)** классифицирует переменные на q-типы:
- **P (predicate)** — переменная используется только в предикатах.
- **M (modified)** — переменная изменяется внутри модуля.
- **C (controlled)** — переменная вычисляется из входных данных модуля.
- **T (tainted)** — входная переменная, не модифицируемая и не входящая в предикат.

```
Q = q₄ + 2·q₃ + 3·q₂ + 0·q₁
```
где q₁=|T|, q₂=|P|, q₃=|C|, q₄=|M|.

**Метрика числа DU-пар** — суммарное количество пар (def, use) для всех переменных в методе.

### 4.2. Граф потока управления для `isGeometricProgression()`

Метод выбран как наиболее сложный (V(G) = 6).

```
Блок 1: [вход]
    if (elements.length <= 1)
           |YES               |NO
           ↓                  ↓
Блок 2: return true     Блок 3: if (elements[0] == 0)
                                    |YES            |NO
                                    ↓               ↓
                            Блок 4: return false   Блок 5: вычислить num, den, g
                                                           num /= g; den /= g
                                                           |
                                                           ↓
                                                   Блок 6: for (i = 1; i < len-1; i++)
                                                           |YES (условие цикла)   |NO
                                                           ↓                       ↓
                                                   Блок 7: if (elements[i] == 0)  Блок 11: return true
                                                           |YES        |NO
                                                           ↓           ↓
                                                   Блок 8: return false  Блок 9: if (проверка ГП !=)
                                                                                  |YES         |NO
                                                                                  ↓             ↓
                                                                          Блок 10: return false  → Блок 6
```

**Параметры графа:**
- Узлов (N) = 11
- Рёбер (E) = 15
- V(G) = E − N + 2 = 15 − 11 + 2 = **6** ✓

**Линейно независимые пути (6 штук):**
1. 1→2 (длина ≤ 1 → true)
2. 1→3→4 (первый элемент = 0 → false)
3. 1→3→5→6→11 (цикл не выполняется ни разу → true)
4. 1→3→5→6→7→8 (elements[i] == 0 → false)
5. 1→3→5→6→7→9→10 (нарушение ГП → false)
6. 1→3→5→6→7→9→6→11 (все проверки пройдены → true)

### 5.3. Метрика Шапена для класса `Sequence`

Классификация переменных метода по q-типам (на примере `Sequence(int, double, int)`):

| Переменная | Изменяется внутри? | Участвует в предикатах? | Тип (q) |
|------------|-------------------|------------------------|---------|
| `first` | Нет | Да (`first == 0`) | **P** |
| `ratio` | Нет | Да (`ratio == 0`) | **P** |
| `count` | Нет | Да (`count <= 0`) | **P** |
| `this.elements` | Да (заполняется) | Нет | **M** |
| `current` | Да (`current *= ratio`) | Нет | **M** |
| `i` | Да (`i++`) | Да (`i < count`) | **M** |

```
q₁ (T) = 0,  q₂ (P) = 3,  q₃ (C) = 0,  q₄ (M) = 3

Q = q₄ + 2·q₃ + 3·q₂ + 0·q₁
Q = 3 + 2·0 + 3·3 + 0 = 3 + 9 = 12
```
