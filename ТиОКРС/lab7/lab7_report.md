## 2. Теоретические сведения

### 2.1. Метрики Холстеда

Метрики Холстеда (Halstead, 1977) основаны на подсчёте **операторов** и **операндов** в исходном тексте программы.

**Базовые величины:**

| Обозначение | Определение |
|-------------|-------------|
| η₁ | Количество **уникальных** операторов |
| η₂ | Количество **уникальных** операндов |
| N₁ | Суммарное число **вхождений** операторов |
| N₂ | Суммарное число **вхождений** операндов |

**Производные метрики (формулы 2.1–2.3):**

```
(2.1)  η = η₁ + η₂                    — словарь программы
(2.2)  N = N₁ + N₂                    — длина программы
(2.3)  V = N × log₂(η)                — объём программы (бит)
```

**Дополнительные метрики:**

```
D = (η₁ / 2) × (N₂ / η₂)             — сложность (difficulty)
E = D × V                              — усилие (effort)
T = E / 18                             — время разработки (секунды)
B = V / 3000                          — оценка числа ошибок
```

**Классификация токенов:**

*Операторы* — ключевые слова управления (`if`, `else if`, `for`, `return`, `throw`, `new`), операторы присваивания (`=`, `/=`, `*=`), арифметические (`+`, `-`, `*`, `%`), сравнения (`==`, `!=`, `<`, `>`, `<=`, `>=`), логические (`||`, `&&`, `!`), тернарный (`?:`), оператор приведения типа (`(int)`, `(long)`, `(Sequence)`), доступ к члену (`.`), индекс массива (`[]`), постфиксный инкремент (`++`), `instanceof`.

*Операнды* — имена переменных, имена методов при вызове, имена классов, числовые литералы, строковые литералы, `null`, `true`, `false`.

## 7. Расчёт метрик Холстеда

### 7.1. Базовые метрики

| Метрика | Значение |
|---------|----------|
| η₁ — уникальных операторов | **29** |
| η₂ — уникальных операндов | **68** |
| N₁ — вхождений операторов | **221** |
| N₂ — вхождений операндов | **273** |

### 7.2. Формулы (2.1) – (2.3)

**(2.1) Словарь программы:**

```
η = η₁ + η₂ = 29 + 68 = 97
```

**(2.2) Длина программы:**

```
N = N₁ + N₂ = 221 + 273 = 494
```

**(2.3) Объём программы:**

```
V = N × log₂(η) = 494 × log₂(97)

log₂(97) = ln(97) / ln(2) = 4,5747 / 0,6931 ≈ 6,600

V = 494 × 6,600 ≈ 3260,4 бит
```

### 7.3. Дополнительные метрики Холстеда

**(2.4) Сложность (Difficulty):**

```
D = (η₁ / 2) × (N₂ / η₂)
D = (29 / 2) × (273 / 68)
D = 14,5 × 4,015
D ≈ 58,2
```

**(2.5) Усилие (Effort):**

```
E = D × V = 58,2 × 3260,4 ≈ 189 755 единиц
```

**(2.6) Расчётное время разработки:**

```
T = E / 18 = 189 755 / 18 ≈ 10 542 сек ≈ 2 ч 56 мин
```

**(2.7) Оценка числа ошибок:**

```
B = V / 3000 = 3260,4 / 3000 ≈ 1,09 ошибки
```

### 7.4. Сводная таблица метрик

| Метрика | Значение | Интерпретация |
|---------|----------|---------------|
| Словарь η | 97 | Умеренно богатый словарь |
| Длина N | 494 | Средняя программа |
| Объём V | 3 260 бит | Умеренная сложность |
| Сложность D | 58,2 | Достаточно сложная реализация |
| Усилие E | 189 755 | Среднее усилие на реализацию |
| Время T | ≈ 2 ч 56 мин | Оценочное время кодирования |
| Ошибки B | ≈ 1,09 | Ожидается ~1 дефект |

## 8. Анализ результатов

**Наиболее сложные методы по числу токенов:**

| Метод | N₁ + N₂ | Доля от общего |
|-------|---------|----------------|
| `isGeometricProgression()` | 37 + 48 = **85** | 17,2% |
| `Sequence(int, double, int)` | 26 + 28 = **54** | 10,9% |
| `getType()` | 18 + 27 = **45** | 9,1% |
| `isArithmeticProgression()` | 20 + 24 = **44** | 8,9% |

**Выводы по метрикам:**

1. **Словарь η = 97** — класс использует разнообразный набор операторов (29) и операндов (68), что отражает богатую семантику класса.

2. **Объём V ≈ 3260 бит** соответствует программе умеренной сложности — реализация нескольких алгоритмических методов в рамках одного класса.

3. **Сложность D ≈ 58** — относительно высокая, обусловлена большим числом повторных использований операндов (`elements`, `i`, `num`, `den`).

4. **Оценка ошибок B ≈ 1** хорошо согласуется с практикой: для программ такого размера ожидается 1–2 дефекта при первичной разработке. Применение TDD в ЛР №5 позволило выявить и устранить их в процессе разработки.
