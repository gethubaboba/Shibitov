## 2. Теоретическая часть

### 2.1. Датасет Boston Housing

Boston Housing - классический учебный датасет для задач регрессии. В нем описаны районы Бостона и пригородов. Каждая строка соответствует одному району, а признаки описывают социальные, экологические, транспортные и экономические характеристики.

Целевая переменная:

- `MEDV` - медианная стоимость жилья в районе, измеряется в тысячах долларов.

Основные признаки:

| Признак | Смысл |
|---|---|
| `CRIM` | уровень преступности на душу населения |
| `ZN` | доля земли под жилые участки большой площади |
| `INDUS` | доля неторговых промышленных площадей |
| `CHAS` | индикатор близости к реке Charles |
| `NOX` | концентрация оксидов азота, показатель загрязнения |
| `RM` | среднее число комнат в доме |
| `AGE` | доля старых домов |
| `DIS` | расстояние до деловых центров |
| `RAD` | доступность радиальных автомагистралей |
| `TAX` | налоговая ставка |
| `PTRATIO` | соотношение учеников и учителей |
| `B` | демографический показатель из исходного набора |
| `LSTAT` | доля населения с низким социальным статусом |

Задача относится к регрессии, потому что нужно предсказывать числовое значение `MEDV`, а не класс.

### 2.2. NumPy

NumPy - библиотека для быстрых численных вычислений. Ее главный объект - многомерный массив `ndarray`. В отличие от обычных списков Python, массивы NumPy хранят элементы одного типа и позволяют выполнять операции сразу над целыми массивами.

Например, выражение:

```python
X.T @ X
```

означает матричное умножение транспонированной матрицы признаков на саму матрицу признаков. Такие операции лежат в основе линейной алгебры и машинного обучения.

В лабораторной NumPy используется для ручной реализации линейной регрессии через нормальное уравнение:

```python
beta = (X.T @ X)^(-1) @ X.T @ y
```

Эта формула находит такие коэффициенты модели, при которых сумма квадратов ошибок минимальна.

### 2.3. Pandas

Pandas - библиотека для работы с табличными данными. Основной объект - `DataFrame`, то есть таблица со строками и именованными столбцами.

Pandas удобен для:

- загрузки данных из CSV и других форматов;
- просмотра первых строк через `head()`;
- проверки типов данных через `dtypes`;
- поиска пропусков через `isnull().sum()`;
- расчета статистик через `describe()`;
- группировок, фильтраций и вычисления корреляций.

В данной работе Pandas является основным инструментом первичного анализа.

### 2.4. Sklearn

Scikit-learn - библиотека машинного обучения. Она предоставляет готовые реализации моделей, предобработки и метрик качества.

В лабораторной используются:

- `StandardScaler` - нормализация признаков;
- `LinearRegression` - линейная регрессия;
- `train_test_split` - разделение данных на обучающую и тестовую выборки;
- `r2_score`, `mean_absolute_error`, `mean_squared_error` - метрики качества;
- `KMeans` - кластеризация.

Главная идея Sklearn - единый интерфейс:

```python
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
```

Метод `fit()` обучает модель, а `predict()` строит прогноз.

### 2.5. Корреляция

Корреляция показывает направление и силу линейной связи между двумя числовыми признаками. Обычно используется коэффициент корреляции Пирсона.

Значения:

- близко к `1` - сильная положительная связь;
- близко к `-1` - сильная отрицательная связь;
- близко к `0` - линейная связь слабая или отсутствует.

Для Boston Housing ожидаемо:

- `RM` положительно связан с `MEDV`: больше комнат - выше цена;
- `LSTAT` отрицательно связан с `MEDV`: выше доля бедного населения - ниже цена;
- `PTRATIO` отрицательно связан с `MEDV`: высокая нагрузка на учителей часто связана с более низкой привлекательностью района.

Важно: корреляция не доказывает причинность. Она показывает статистическую связь, но не объясняет все причины изменения цены.

### 2.6. Нормализация признаков

Признаки в датасете имеют разные масштабы. Например, налоговая ставка `TAX` может измеряться сотнями, а `NOX` - долями единицы. Если модель или алгоритм чувствительны к масштабу признаков, большие по численному диапазону признаки могут доминировать.

`StandardScaler` преобразует признаки так, чтобы:

- среднее значение стало примерно `0`;
- стандартное отклонение стало примерно `1`.

Формула стандартизации:

```text
z = (x - mean) / std
```

Для линейной регрессии без регуляризации нормализация не всегда обязательна, но она полезна для корректного сравнения признаков и особенно важна для KMeans.

### 2.7. Выбросы и Z-score

Выброс - наблюдение, которое сильно отличается от основной массы данных. Выбросы могут искажать средние значения, корреляции и коэффициенты модели.

Z-score показывает, на сколько стандартных отклонений значение отличается от среднего:

```text
z = (x - mean) / std
```

Если `abs(z) > 3`, значение часто считают потенциальным выбросом. В работе удаляются строки, где хотя бы один признак имеет Z-score больше 3.

Плюс такого подхода - простота. Минус - он предполагает примерно нормальное распределение признаков, что не всегда выполняется.

### 2.8. Линейная регрессия

Линейная регрессия строит модель вида:

```text
MEDV = b0 + b1 * RM + b2 * LSTAT + b3 * PTRATIO
```

где:

- `b0` - свободный член;
- `b1`, `b2`, `b3` - коэффициенты признаков.

Коэффициент показывает, как в среднем меняется целевая переменная при увеличении соответствующего признака на единицу, если остальные признаки фиксированы.

Ожидаемые знаки коэффициентов:

- `RM` - положительный;
- `LSTAT` - отрицательный;
- `PTRATIO` - отрицательный.

### 2.9. Метрики качества регрессии

Для оценки модели используются несколько метрик.

`R²` показывает, какую долю вариации целевой переменной объясняет модель. Чем ближе к `1`, тем лучше. Значение около `0.65-0.70` для простой модели на трех признаках является приемлемым учебным результатом.

`MAE` - средняя абсолютная ошибка:

```text
MAE = mean(abs(y_true - y_pred))
```

Она показывает средний размер ошибки в исходных единицах измерения.

`RMSE` - корень из средней квадратичной ошибки:

```text
RMSE = sqrt(mean((y_true - y_pred)^2))
```

RMSE сильнее штрафует большие ошибки, поэтому полезна, когда крупные промахи особенно нежелательны.

### 2.10. Нормальное уравнение NumPy

Линейную регрессию можно решить аналитически без итерационного обучения. Для этого используется формула:

```text
beta = (X^T X)^(-1) X^T y
```

Здесь:

- `X` - матрица признаков;
- `y` - вектор целевых значений;
- `beta` - вектор коэффициентов;
- `X^T` - транспонированная матрица.

Чтобы посчитать свободный член, к матрице `X` добавляют столбец единиц. Тогда первый коэффициент `beta[0]` становится `Intercept`.

Если результат NumPy совпадает со Sklearn, значит ручная реализация корректна.

### 2.11. KMeans

KMeans - алгоритм кластеризации без учителя. Он разбивает объекты на `k` групп так, чтобы объекты внутри одной группы были похожи друг на друга.

Алгоритм работает так:

1. Случайно выбираются центры кластеров.
2. Каждая точка относится к ближайшему центру.
3. Центры пересчитываются как средние значения точек в кластере.
4. Шаги повторяются до стабилизации.

В лабораторной кластеризация применяется к признакам `CRIM`, `NOX`, `DIS`, чтобы выделить типы районов по преступности, загрязнению и удаленности от центра.

### 4.2. Первичный анализ

Первичный анализ отвечает на простые, но важные вопросы:

- сколько данных доступно;

### 4.3. Предобработка

Нормализация делает признаки сопоставимыми по масштабу. Это особенно важно для KMeans, потому что расстояния между точками считаются в пространстве признаков. Без нормализации признак с большим диапазоном мог бы полностью определять кластеры.

Бинарный признак `is_high_value` не нужен для линейной регрессии из задания напрямую, но полезен как пример feature engineering: из непрерывной цены выделяется индикатор дорогого жилья.

Удаление выбросов снижает влияние аномальных районов на коэффициенты модели.

### 4.4. Визуализация

Тепловая карта корреляций помогает увидеть не только связь признаков с `MEDV`, но и связи признаков между собой. Это важно, потому что сильная корреляция между признаками может приводить к мультиколлинеарности.

Scatter plot `RM` vs `MEDV` показывает положительную зависимость: больше комнат обычно означает более дорогой дом.

Scatter plot `LSTAT` vs `MEDV` показывает отрицательную и, вероятно, нелинейную зависимость. Поэтому простая линейная модель может недоиспользовать информацию в `LSTAT`.

### 4.5. Линейная регрессия

Модель обучается на очищенных данных и использует три признака:

```python
features = ['RM', 'LSTAT', 'PTRATIO']
```

Эти признаки выбраны осмысленно:

- `RM` отражает качество/размер жилья;
- `LSTAT` отражает социально-экономический уровень района;
- `PTRATIO` косвенно связан с качеством школ.

После обучения модель выдает коэффициенты. Их нужно не просто вывести, а интерпретировать: знак коэффициента показывает направление связи, а модуль - силу влияния в рамках масштаба признака.

### 4.6. Ручная реализация через NumPy

Ручная реализация важна для понимания, что Sklearn не является «магией». Линейная регрессия сводится к задаче минимизации квадратичной ошибки. Нормальное уравнение дает аналитическое решение.

Сравнение коэффициентов NumPy и Sklearn показывает, что обе реализации решают одну и ту же математическую задачу. Малые отличия возможны из-за численной точности.

### 4.7. Кластеризация

KMeans решает другую задачу: не предсказывает цену, а группирует районы по похожести. В работе используются `CRIM`, `NOX`, `DIS`, потому что они описывают среду района:

- безопасность;
- экологию;
- удаленность.

После кластеризации полезно посмотреть среднюю `MEDV` по каждому кластеру. Это связывает результаты кластеризации с экономическим смыслом.
